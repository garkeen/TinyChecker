## 核心语法设计

核心语法是内核直接处理的语法形式。所有表面语法在繁饰后被翻译为核心语法。

### 核心项

**定义**（核心项）：核心项 `CTerm` 只有五种构造。

- **类型宇宙**（`CType`）：类型的类型，相当于表面语法的 Type/Prop。
- **变量**：`CVar(k)` 表示局部变量（de Bruijn 索引），`CGlobal(name)` 表示全局变量。相当于表面语法的 Atomexpr。
- **依赖函数类型**（`CPi`）：以 `CPi(A, B)` 记入信息，输入变量类型为 A，返回类型为 B，默认所有函数类型都是依赖函数类型。相当于表面语法的 `(X:A)->B`。由于使用 de Bruijn 索引，不需要记录 X 了。
- **Lambda 抽象**（`CLam`）：以 `CLam(A, body)` 记入信息，输入变量类型为 A，函数体为 B。相当于表面语法的 `\(X:A)=>B`。由于使用 de Bruijn 索引，不需要记录 X 了。
- **函数应用**（`CApp`）：以 `CApp(f, a)` 记入信息，指 f 应用 a。

### 核心声明

**定义**（核心声明）：核心声明 `CDecl` 分为无值变量和有值变量两大类，用 `kind` 标签区分全局条目类别。

无值变量声明——有类型但没有值，不能进行 Delta 规约：

| kind | 说明 | 来源 |
|------|------|------|
| `type_ctor` | 类型构造器 | `inductive` / `sum` / `product` |
| `data_ctor` | 数据构造器 | 同上 |
| `axiom` | 公理 | `axiom` |
| `rec` | 归纳子 | `inductive` 自动生成 `D.rec` |
| `elim` | 消去子 | `sum` / `product` 自动生成 `D.elim` |

有值变量声明——有值，可以进行 Delta 规约：

| kind | 说明 | 来源 |
|------|------|------|
| `var` | 变量 | `var` 声明 |
| `claim` | 断言 | `claim` 声明 |
| `fun` | 函数 | `fun` 声明 |
| `theorem` | 定理 | `theorem` 声明 |
| `projection` | 投影函数 | `product` 自动生成 `D.f1 ... D.fn` |

### 特殊求值规则

对 `CApp(f, a)` 规约的时候，f 不一定必须是 `CLam(A, body)` 然后触发 Beta 规约，也可以是所谓的归纳子（消去子共享和归纳子一样的求值逻辑），这个时候需要对这个应用做特殊处理。归纳子（`D.rec`）或消去子（`D.elim`）参数满足，且最后一个参数（scrutinee）拥有构造器头时，触发 Iota 规约。Iota 区别于普通的 Beta/Delta 规约：它不是简单的函数代入，而是根据 scrutinee 的构造器选择对应分支并代入构造器的字段值，以及归纳假设。

## de Bruijn 表示

### 作用域与 alpha 等价

**定义**（作用域）：一个表达式内可以引用的所有变量是这个表达式的作用域。一般来说，我们会区分局部作用域和全局作用域。局部作用域保存表达式可见的局部变量，全局作用域保存表达式可见的全局变量。

Decl 中定义的变量就是全局变量，比如：

```
var x:Nat = zero;
var f:Nat->Nat = \(n:Nat)=> succ n;
```

x 和 f 就是所谓的全局变量。Expr 中通过 Pi 头和 Lambda 头定义的变量则是局部变量，比如：

```
var f:(a:Nat)->(b:Nat)->(p:Eq Nat a b)->Eq Nat b a = \(x:Nat)=>\(y:Nat)=>\(q:Eq Nat x y)=> eqTrans Nat Nat a b q;
```

f 的类型表达式中，其返回类型表达式 `Eq Nat b a` 可以引用之前 Pi 头引入的变量 `a b p`，这些便是局部变量。同理 f 的值表达式中的函数体 `eqTrans Nat Nat a b q` 可以引用之前 Pi 头引入的变量 `x y q`。然而考虑下面的问题：

```
\(x:Nat) => x 和 \(y:Nat) => y
```

两者是一个东西吗？两者语义上完全相同（恒等函数），但名字不同。这意味着，如果直接用语法结构比较两者，会得到不相等的结果。

**定义**（alpha 等价）：如果两个函数对输入变量进行恰当的重命名之后语法结构完全相同，那么这两个函数是 alpha 等价的。

**定义**（作用域遮蔽）：全局作用域不能存在同名变量，局部却可以，此时更深层次的同名变量会覆盖浅层次的。比如 `(x:Type)=>\(x:x)=>x` 是合法的定义，最终 Body 的第二个输入 x。而第二个输入 x 的类型是第一个输入 x。

### de Bruijn 索引

我们当然可以在类型检查的时候使用重命名动态的判断两个函数是否相等，然而，我们可以用更优雅的方式把重命名提前到类型检查之前。之前的例子我们注意到，名字其实不重要，名字所处的位置才是重要的。比如：

```
\(F:A->A->A)=>\(x:A)=>\(y:A)=>F x y
\(F:A->A->A)=>\(y:A)=>\(x:A)=>F x y
```

两者内部表达式相同，然而语义却不同，这关键在于 x 和 y 的位置不一样。因此，我们可以用位置代替变量名，这里的位置就是所谓的 de Bruijn 索引。

**定义**（de Bruijn 索引规则）：

- 每个 Lambda 或 Pi 引入的变量会在当前局部上下文栈增加一个槽位。
- 变量用自然数 `k` 表示，`k` 是当前位置到其绑定变量之间跨越的 Lambda 或 Pi头数量。
- 最近的引入变量对应 `k = 0`，次近对应为 `k = 1`，依此类推。

示例：

```
\(x:Nat) => x              →   \(Nat) => 0
\(x:Nat) => \(y:Nat) => x  →   \(Nat) => \(Nat) => 1
\(x:Nat) => \(y:Nat) => y  →   \(Nat) => \(Nat) => 0
```

有了 de Bruijn 索引，两个函数语法相等就会自动推出 alpha 等价。
