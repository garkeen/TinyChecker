# 摘要

近年来，自然语言大模型不断迭代进化，人们开始尝试使用大模型进行编程和数学证明。由于基于概率生成的大模型不可避免的拥有幻觉，如何验证大模型生成证明的正确性是一个值得研究的难题。为了解决这个问题，研究人员开始使用定理证明助手，比如 Google DeepMind 团队 AlphaProof 项目就是基于 Lean 反馈的强化学习训练而成。然而，定理证明助手作为一个极其专业化的软件，对使用者的数学基础和编程理论基础要求较高，而且学习曲线陡峭，如何用最小成本让初学者快速上手定理证明助手，是一个值得研究的课题。

本课题尝试从理论和实践两个角度，研究基于依赖类型论的现代定理证明助手如何工作，并设计一个教学式的定理证明助手。

理论上，我们探讨了形式化数学的发展史，数学为何能被形式化，以及何为逻辑，何为证明；又探讨了 Lambda 演算，以及在此基础上添加各种类型系统的变体；最后，我们谈论了柯里-霍华德同构，如何把类型系统解释为直觉主义逻辑，命题如何对应类型，证明如何对应程序。

实践上，我们探索了如何实现一个最小的定理证明助手，供理论研究和教学展示。我们借鉴了 Lean Kernel 的开源实现，并做了简化：核心语言只有依赖函数类型和归纳类型，并提供一些表面语法糖，放弃了类型宇宙和策略等高级特性。我们编写了一些测试案例对项目的可行性和正确性进行了测试。

本课题的创新点在于，证明了只要实现依赖函数类型和归纳类型的 Lambda 演算系统就能够胜任现代定理证明器的大部分工作了；此外还设计了一个友好的 GUI 用来展示柯里-霍华德同构。

本课题探讨的内容详实，任何人都可以尝试基于这些内容来实现自己的简单定理证明助手，希望为想要学习定理证明助手的爱好者提供帮助，并推动定理证明助手的普及。

关键词：类型论；依赖类型；定理证明助手

# ABSTRACT

In recent years, large language models have been continuously iterating and evolving, and people have begun attempting to use them for programming and mathematical theorem proving. Since large models based on probabilistic generation inevitably suffer from hallucinations, how to verify the correctness of proofs generated by these models is a challenging problem worth studying. To address this issue, researchers have started using proof assistants; for example, Google DeepMind's AlphaProof project was trained through reinforcement learning based on feedback from Lean. However, as highly specialized software, proof assistants demand a relatively high level of mathematical and programming theory foundations from users and have a steep learning curve. How to enable beginners to quickly get started with proof assistants at minimal cost is a topic worth investigating.

This project attempts to study, from both theoretical and practical perspectives, how modern proof assistants based on dependent type theory work, and to design a teaching-oriented proof assistant.

Theoretically, we explore the history of formalized mathematics, why mathematics can be formalized, and what logic and proof are; we then discuss lambda calculus and the variants extended with various type systems; finally, we discuss the Curry-Howard isomorphism, how type systems can be interpreted as intuitionistic logic, how propositions correspond to types, and how proofs correspond to programs.

Practically, we explore how to implement a minimal proof assistant for theoretical research and pedagogical demonstration. We draw on the open-source implementation of Lean's kernel and simplify it: the core language includes only dependent function types and inductive types, provides some surface-level syntactic sugar, and omits advanced features such as type universes and tactics. We wrote some test cases to test the feasibility and correctness of the project.

The innovation of this project lies in demonstrating that a lambda calculus system implementing only dependent function types and inductive types is sufficient to handle most of the work of a modern theorem prover; furthermore, a user-friendly GUI was designed to illustrate the Curry-Howard isomorphism.

The content explored in this project is detailed and substantial. Anyone can try to implement their own simple proof assistant based on these contents. We hope to provide help to enthusiasts who want to learn proof assistants and to promote the popularization of proof assistants.

Keywords: Type theory; Dependent types; Proof assistant
