## GUI

本系统提供了一个面向教学的表达式观察器与证明可视化工具，基于 PySide6 构建。界面左侧为源码编辑器，右侧通过标签页切换「上下文环境」与「证明视图」。

### 工作模式

GUI 提供分析模式与编译模式，通过分段式按钮切换。两种模式均支持光标驱动的表达式检查。

**分析模式** 执行词法分析、语法分析与 GUI 专用繁饰探针，在抵达目标表达式时立即返回当前上下文快照，不触发完整类型检查。适合日常编码时的快速上下文观察。

**编译模式** 开放「类型检查」按钮。用户手动点击后，系统执行完整的核心繁饰与类型检查流水线，通过后自动生成证明树并填充到证明视图标签页。结果缓存至模式切换。

### 基于光标的聚焦模型

光标位置先映射为基于 token 的焦点：命中 token 时为 $\text{TokenFocus}(i)$，落在间隙时为 $\text{GapFocus}(i)$。语法分析阶段为每个表达式节点记录 token 区间 $[l, r)$，GUI 在所有覆盖当前焦点的表达式中选取区间长度最小者作为聚焦表达式。用户可通过「向外扩展」上升至父表达式，「回到最小」重置。

表达式确定后，编辑器以背景色标记其 token 区间，上下文环境同步刷新。

### 语法高亮

编辑器内建语法高亮，按表面语法词法规则着色：关键字（`var`/`fun`/`inductive`/`match` 等）为蓝紫色粗体，类型关键字（`Type`/`Prop`）为青色，注释为灰色斜体，构造器（`|` 后标识符）为深绿色，运算符与分隔符为暗金色。

### 上下文环境

聚焦表达式后，右侧上下文环境标签页以卡片式布局展示：

- **表达式头**：当前聚焦的表达式文本；
- **局部上下文**：当前作用域内可见的局部绑定名及其类型，从近到远排列；
- **分支信息**（位于 `match`/`case` 内时）：当前构造器名、分支字段、归纳假设；
- **全局上下文**：已定义的全局名及其类别标签（类型构造器、数据构造器、归纳子等）。

上下文信息由 GUI 专用繁饰探针生成，复用主繁饰器的名字解析与约束检查逻辑。

### 证明可视化

类型检查通过后，系统遍历全部核心声明，对每个 `theorem` 与 `claim` 调用证明树构造器。构造器递归分析证明项结构：Lambda 抽象对应"引入假设"，归纳子应用对应"结构归纳法"，消去子应用对应"分类讨论"，函数应用按类型信息展开为逐步推导，每个分支显示其字段与归纳假设。

证明树以 HTML 渲染，不同语义类型的节点以左边框颜色区分：归纳法为绿色，分类讨论为紫色，普通推导为灰色。证明视图为全局视图，展示文件中全部定理，但源码光标落在某定理声明内时，视图自动滚动至该定理的证明块。

证明可视化将柯里-霍华德同构以直观方式呈现：源码中的 `theorem` 对应自然语言定理陈述，`match` 对应归纳法或分类讨论，整个证明项对应完整的证明过程。
